Régression logistique¶

Mise en situation¶

Je travaille pour un organisme de crdit, je dois deviner rapidement la probabilit qu'une personne soit un "bon payeur" ou un "mauvais payeur" en fonction de son profil.

Je ne vois pas trop comment la rgression linaire pourrait m'aider...

Régression logistique¶

C'est quoi ?¶

Jusque-l, nous avons travaill sur un modle qui nous donne une valeur quantitative (un nombre) en sortie. La classification est un autre type de problme, qui ncessite un autre type de modle - ou en tout cas une adaptation de ce qu'on a dj fait.

On dispose aujourd'hui

  • d'un modle linaire : $f_\theta(x) = x^T.\theta$, qui nous renvoie un rel
  • d'une fonction de cot $J(\theta) = \frac{1}{m}\sum_{i=1}^m{(f_\theta(x_i)-y_i)^2}$
  • du gradient $\frac{dJ}{d\theta}$ de cette fonction, qui nous permet soit de trouver une solution directe, soit d'appliquer une descente de gradient

On souhaite dsormais disposer, pour une catgorie d'objets identifier :

  • d'un modle qui nous renvoie 1 (objet dans la catgorie) ou 0 (hors catgorie) - ou la rigueur une valeur entre 0 et 1, une probabilit d'appartenance la catgorie
  • d'une fonction de cot qui marche bien pour ce modle
  • du gradient associ

La fonction sigmoïde¶

La fonction sigmode est dfinie par $f(x) = \frac{1}{1+e^{-x}}$. Graphiquement a ressemble a :

On remarque que

  • pour $x=0$, $sigmoid(x)=0.5$
  • pour $x>0$, $sigmoid(x)>0.5$
  • pour $x<0$, $sigmoid(x)<0.5$

On pourrait utiliser cette fonction dans notre modle, combin avec un modle linaire : on commence par calculer une valeur numrique relle, et on y applique la sigmode pour obtenir une probabilit.

Plus la valeur calcule dans la premire tape est suprieure 0 et plus la probabilit est grande; plus elle est infrieure 0 et plus la probabilit est faible.

Le modèle¶

On va donc dfinir un nouveau modle logistique d'aprs la nouvelle fonction $f$:

$f_\theta(x) = sigmoid(\sum_{i=0}^n{(\theta_i.x_i)}) = sigmoid(x^T.\theta)$

On va aussi dfinir une fonction de cot $J(\theta)$ : pour cel il faut d'abord savoir ce qu'on appelle une erreur.

Quand $y$, la valeur attendue, vaut 1, on s'attend avoir f(x) le plus prs de 1 possible; et inversement quand $y$ vaut 0 on cherche f(x) le plus proche de 0 possible. La fonction gnralement retenue est :

$J(\theta) = \frac{1}{m}\sum_{i=1}^m{\left[ -y^{(i)}.log\left(f_\theta(x^{(i)})\right) -(1-y^{(i)}).log\left(1-f_\theta(x^{(i)})\right) \right]}$

Quand $y$ vaut 1, on ne considre que le premier terme (l'autre est nul), et il vaut $-log\left(f_\theta(x^{(i)})\right)$ : plus $f_\theta(x^{(i)})$ est proche de 1 et plus l'erreur est faible, et inversement.

Et quand $y$ vaut 0, c'est le second terme qu'on considre, et pareil plus $f_\theta(x^{(i)})$ est proche de 0, plus l'erreur est faible et inversement.

On a l'habitude aussi de sortir le signe ngatif :

$J(\theta) = -\frac{1}{m}\sum_{i=1}^m{\left[ y^{(i)}.log\left(f_\theta(x^{(i)})\right) +(1-y^{(i)}).log\left(1-f_\theta(x^{(i)})\right) \right]}$

Le gradient¶

Dans un premier temps, calculons la drive de la sigmode $s(x)$. Vous pourrez vrifier le rsultat suivant : $s'(x) = (1-s(x)).s(x)$

La drive partielle $\frac{\partial f_\theta}{\partial \theta_i}$ peut aussi se calculer facilement maintenant : $\frac{\partial f_\theta}{\partial \theta_i} = (1-f_\theta(x)).f_\theta(x).x_i$, il s'agit de la drive d'une composition de fonction (FoG)' = F'oG.G'

On poursuit avec les logarithmes de $f$ ou de $1-f$, et on termine avec $J$ : $\frac{\partial J}{\partial \theta_i} = \frac{1}{m}\sum_{z=1}^m{\left[(f_\theta(x^{(z)})-y^{(z)}).x_i^{(z)}\right]}$

La formule finale matricielle est : $\frac{dJ}{d\theta} = \frac{1}{m}(X^T.(f_\theta(X)-Y)$

Remarque intressante: la drive du gradient est assez similaire ce qu'on a vu pour la rgression linaire. Dans la rgression linaire, on avait $\frac{dJ}{d\theta} = \frac{1}{m}X^T.(X.\theta-Y)$, or $X.\theta = f_\theta(X)$ dans le cadre de la rgression linaire.

def f(x, theta):
    return sigmoid(np.dot(x, theta))

def solve_model(x, y, alpha=0.1, iterations=1000):
    m = x.shape[0]
    #On ajoute toujours la colonne de 1
    Xtmp = np.concatenate([np.ones((m,1)), x], axis = 1)
    
    theta = np.ones((Xtmp.shape[1], 1))
    for i in range(iterations):
        evaluations = f(Xtmp, theta)
        errors = evaluations - y
        gradient = np.dot(Xtmp.T, errors) / m
        theta = theta - alpha*gradient
    return theta

theta = solve_model(X, Y)

def plot_bounds(model, X, Y, draw_box):
    h=0.01
    mesh_x, mesh_y = np.meshgrid(
        np.arange(X[:,0].min() - 1, X[:,0].max() + 1, h),
        np.arange(X[:,1].min() - 1, X[:,1].max() + 1, h))
    Z = model(np.c_[np.ones((mesh_x.ravel().shape[0], 1)), mesh_x.ravel(), mesh_y.ravel()])
    Z = Z.reshape(mesh_x.shape)
    draw_box.contourf(mesh_x,mesh_y,Z, cmap=plot.cm.Spectral)
    draw_box.scatter(X[:,0], X[:,1], c=Y, cmap=plot.cm.Spectral)

def sigmoid(x): return 1/(1+np.exp(-x))
def f(x, theta): return sigmoid(np.dot(x, theta))
def predict(x, theta, mus, sigmas):
    x_mod = (x - mus)/sigmas
    x_mod = np.concatenate([np.ones((x.shape[0], 1)), x_mod], axis = 1)
    return f(x_mod, theta)

def cost(x, y, theta):
    y_p = f(x, theta)
    return -np.average(y * np.log(y_p) + (1-y) * np.log(1-y_p))

def train_model(X, Y, alpha = 0.01, iterations = 1000):
    m = X.shape[0]
    
    #normalization
    mus = np.mean(X, axis = 0)
    sigmas = np.std(X, axis = 0)
    
    Xtmp = np.concatenate([np.ones((m, 1)), (X-mus)/sigmas], axis = 1)
    
    theta = np.random.rand(Xtmp.shape[1], 1) * 0.01
    costs = []
    for i in range(iterations):
        if i%100 == 0: costs.append(cost(Xtmp,Y,theta))
        evaluations = f(Xtmp, theta)
        errors = evaluations - Y
        grad = np.dot(Xtmp.T, errors) / m
        theta = theta - alpha * grad
    return theta,costs, mus, sigmas
